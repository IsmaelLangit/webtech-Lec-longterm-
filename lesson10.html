CHAPTER 10: OWASP Top Ten Projects

OWASP Top Ten Projects

OWASP –The Open Web Application Security Project is an organization providing unbiased and practical information about computer and internet applications. Every 3 years they are releasing the Top Ten most dangerous Web Application security vulnerabilities and providing effective solutions.

Injection

Injection flaws, it occurs when an intruder send an untrusted data to an interpreter as a part of command or query. the intruder enters an SQL statement into an entry field and the input is not filtered for escape characters.

Broken authentication and session management

When authentication functions related to the application are not implemented correctly which will allow hackers to compromise passwords specially session ID’s. Threat agents uses leaks or flaws in the authentication or session management functions.

Cross site scripting (XSS)

Attackers can execute scripts in the browser of the victim and uses the script to hijack session ID’s, hijack the user browser, or deface the website.Often called session stealing or cookie stealing.

Broken access control

Attackers who are authorized changes the value of the parameters in order to gain access on resources that they are unauthorized to access.

p>
Security Misconfiguration

Default accounts and misconfigurations are the flaws or vulnerabilities, attackers can access misconfigured accounts and default accounts. They can access those directories and gain unauthorized access. Such flaws result in a compromised system.

p>
Sensitive data exposure

Attackers may steal or modify such weakly protected data to conduct credit card fraud, identity theft, or other crimes. Sensitive data deserves extra protection such as encryption at rest or in transit, as well as special precautions when exchanged with the browser.

p>
Insufficient attack protection

The majority of applications and APIs lack the basic ability to detect, prevent, and respond to both manual and automated attacks. Attack protection goes far beyond basic input validation and involves automatically detecting, logging, responding, and even blocking exploit attempts.
