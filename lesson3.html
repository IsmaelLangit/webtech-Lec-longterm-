LESSON 3: HTTP Resource Addressing




HTTP resources are identified unding URIs (RFC 3986), or, more specifically, HTTP URLs.

  • scheme (http or https)
  • authority
    • user information or authentication credentials (deprecated).
    • host
      • domain name (resolved to an IP address using DNS) of the server where the resource resides (or will be created).
    • port number
  • path to resource
    • (resolved relative to the document root on the server) ("/" or "\")
    • may refer to a static or dynamic resource
  • query
    • typically provided as key = value pairs, with ampersand (&) separators between key/value pairs.may be URL - encoded.
  • fragment identifier (#) typically used in the browsers end. sometimes called a bookmark
    • Examples:
    • http://usr:pwd@server.org:81/info/profile.php?id 1234#adr
    • https://www.google.com
