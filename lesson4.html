CHAPTER 4: CSS (Cascading Style Sheets)

CSS (CASCADING STYLE SHEETS)

Managed also by W3 Consortium

  • Language used to specify the presentation aspects (e.g layout and formatting of structurally marked up documents)
  • Developed by Hakon Wium Lie (CHSS) and Bert Bos (SSP)
  • Versions
    • CSS 1 (December 17 1996) revised on April 11 2008
    • CSS 2 (monolithic specification or one stop shop)
    • CSS 3 (no monolithic specification, new addition such as Media queries, animations, transforms)

CSS SELECTORS

Structure used as a condition in a CSS rule to determine which elements in the document tree are matched by the selector and are thus targeted by the formatting specified in the CSS rule declaration block. The matched elements are called the subjects of the selector.

Selector-syntax

Chain of one or more sequences of simple selectors separated by combinators with one pseudo-element possibly appended to the last sequence

Example

Dir#abc>p.xyz[title]+span:last-of-type::after

//Dir (type)

//Dir#abc (sequence child of)

//> (combinator)

//P (paragraph)

//.xyz (class)

//[title] (title attribute)

//P.xyz (sequence adjacent)

//[title]+span combinatory)

//:last-of-type (pseudo class)

//::after (pseudo element)

Sequence of Simple Selectors

  • Chain of simple selectors not separated by combinators
  • Always starts with a type selector or a universal selector
  • Cannot contain other type selectors or universal selectors

Group of Selectors

Comma-separated list of selectors representing the union of all elements selected by each of the selectors in the list. (e.g h1,h2,h3 – group selectors separated by comma)

Simple/Basic Selectors
  • Type selector – type of element you are targeting (all the elements)
  • Universal – represented by the asterisk (*) matches everything
  • Attribute
    • [attr] – class attribute (name of a string)
    • [attr] – class attribute (name of a string)
    • [attr=value] – must be exactly the same
    • [attr~=value] – this value is just one value in a set of values that are space separated. Or one that is exactly match
    • [attr|value] – used to target the language attribute (lang|=en)
    • [attr^=value] – beginning value (CSS3)
    • [attr$=value]- end value (CSS3)
    • [attr*=value] value that is found anywhere (CSS3)

Class selectors

full stop (“.”) and a class value

Syntax

.x // universal selector

p.x //type selector

.sampleClass {

 margin : 20px;

 background-color : white;

 color : black;

}

ID selector

Pound sign (#) and ID value

Syntax

#a

#sampleID {

 padding : 0 50px;

 background-color : white;

}

Pseudo-classes

Implicit class that is associated with the inherited classes or mark up (fake class)

  • Dynamic pseudo class
    • Link pseudo classes
      • :link
      • :visited
  • User action pseudo classes
    •  :hover
    • :active
    • :focus
  • Target pseudo class (targeting element)
    • :target (CSS3)
  • Language pseudo class
    • :lang()
  • UI element states pseudo classes
    • :enabled (CSS3)
    • :disabled (CSS3)
    • :checked (CSS3)
    • :indeterminate (CSS3)
  • Structural pseudo classes
    • :root (CSS3)
    • :first-child (CSS3)
    • :last-child (CSS3)
    • :only-child (CSS3)
    • :nth-child() (numbers and literals – 2n+1) (CSS3)
    • :nth-last-child() (count on the last child –reverse) (CSS3)
    • :first-of-type (CSS3)
    • :last-of-type (CSS3)
    • :only-of-type (CSS3)
    • :nth-of-type()(CSS3)
    • :nth-last-of-type()(CSS3)
    • :empty (element that doesn’t contain anything) (CSS3)
  • Negation pseudo class
    • :not() – simple selector (negates the logic of the selector)
  • Combinators
    • Descendant combinators (whitespace, ie. Space, tab, linefeed, carriage return, form feed)
      • Child combinatory (>) direct descendant
      • Siblings combinators – at the same level
        • Adjacent sibling combinators (+)
        • General sibling combinators (~)
  • Pseudo-elements
    • ::first-letter (CSS3) :first-letter
    • ::first-line (CSS3) :first-line
    • ::before (CSS3) :before
    • ::after (CSS3) :after
CSS RULE PRECEDENCE

  1. By origin and importance

    • User agent important declaration
    • User important declaration
    • Author important declaration
    • Author normal declaration
    • User normal declaration
    • User agent normal declaration

      • Example

      p {

       color = red !important;

      }

      p {

       color = green;

      }

  2. By specificity

    • Inline style
    • example
    • Number of ID selectors
    • Number of class selectors, attributes selectors, and pseudo classes
    • Number of type selectors and pseudo-element

  3. By order

    • The later the declaration the higher the priority

      • Example

      h1 {

       color : red

      }

      h1 {

       color : green

      }

      //h1 will be color green
