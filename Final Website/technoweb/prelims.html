The Internet

Internet is short for Internetwork, whereas, internetworking is the process of connecting a single network with other networks. Network is defined as the connection of devices together to communicate and share resources where the connection may either be wired or wireless. Internet is a network of networks that use the Internet Protocol Suite to link the devices worldwide. Protocols are communication standards that devices will agree on.

1965

Two computers at MIT Lincoln Lab communicate with one another using packet-switching technology.

1968

BBN unveils the final version of the Interface Message Processor specifications. BBN wins ARPANET contract.

1972

BBN’s Ray Tomlinson introduces network email. The Internetworking Working Group forms to address need for establishing standard protocols.

1973

Global networking becomes a reality as the University College of London and Royal Radar Establishment connect to ARPANET. The term Internet is born.

1981

The National Science Foundation provided a grant to establish CSNET to provide networking services to university computer scientists.

1982

TCP/IP, emerge as the protocol for ARPANET. TCP/IP remains the standard protocol for the Internet.

1990

Sir Tim Berners-Lee, a scientist at CERN, the European Organization for Nuclear Research, develops HTML.

1991

CERN introduces the World Wide Web to the public.

WAIS(Wide Area Information Servers)

WAIS is the internet system wherein specialized subject databases are created at multiple server locations. It is a client-server text searching system that uses Information Retrievel Service Definition and Protocol Specifications for Library Applications to search index databases on remote computers.

World Wide Web

The world wide web uses the infrastructure provided by the internet. It allows documents to be connected to other documents by hypertext links. These documents are identified as Uniform Resource Locators or URLs and they are interlinked by hypertext links that can be accessed through the internet.

Hypertext Transfer Protocol

  • Application layer communications protocol used to access resources (hypertext/hypermedia) on the World Wide Web.
  • invented by Tim Berners-Lee
  • jointly developed by the W3C and the IETF
      • version history :
    • HTTP 0.9 (1991)
    • HTTP 1.0 (RFC 1945, May 1996)
    • HTTP 1.1 (RFS 2068 Jan 1997, RFC 2116 Jun 1999), RFC 7230 - 7235 (Jun 2014)
        • HTTP 2 (RFC 7540 May 2015)
      • SPDY - google
FUNDAMENTALS OF HTTP
  • HTTP runs on top of TCP/IP, using TCP port 80 by default or TCP port 443 for HTPPS (HTPPS over SSL/TLS)
  • HTTP is based on a client - server achitecture
    • Client, a.k.a. user agents (UA) :
      • web browseras, web crawlers/spider, other end user tools and applications
    • Servers :
      • Origin Servers 
      • Proxy Servers, gateways, tunnels
  • HTTP uses a request-response standard protocol 
    • the clients sends an HTTP request message to the server
    • the servers processes the request and replies with an HTTP response
  • HTTP is a stateless communications protocol
    • Servers do not keep information about clients in-between request
  • HTTP provides support for other functionalities, such as :
    • cache control
    • control media type (MIME) specification
    • language and character set specification
    • content/transfer coding
    • content negotiation
    • client-server protocol negotiations
    • persistent connection
    • request pipelining
    • authentication/authorization
    • etc.
HTTP RESOURCE ADDRESSING
HTTP resources are identified unding URIs (RFC 3986), or, more specifically, HTTP URLs.
Scheme (http or https)
authority
-user information or authentication credentials (deprecated).
-host domain name (resolved to an IP address using DNS) of the server where the resource resides (or will be created).
-port number
Path to resource(resolved relative to the document root on the server) ("/" or "\")
-may refer to a static or dynamic resource
Query
-typically provided as key = value pairs, with ampersand (&) separators between key/value pairs.may be URL - encoded.
Fragment identifier (#)
-typically used in the browsers end.sometimes called a bookmark

Examples:

  • http://usr:pwd@server.org:81/info/profile.php?id 1234#adr
  • https://www.google.com
HTTP Request Message(From Client)
  • Request line (CRLF - terminated line consisting of three space - separated values).
  • Method - What you're requesting the server
  • Request URI - Identifier for resource
  • HTTP protocol version
  • Message Headers
    • HTTP 1.1 requires at least the host request header to be provided.
  • Empty line (CRLF)
  • Message body a.k.a payload(optional)
HTTP Response Message (From Server)
  • Status line(HTTP Protocol Version, Status Code, Reason Phrase)
  • Message headers
  • Empty line (CRLF)
  • Message body a.k.a payload(optional)
  • Status Code
HTTP request methods

Standard Methods:

GET
- Most commonly used HTTP method
- Must be supported by all compliant general purpose servers
HEAD
- MSame as GET except that the entity is not included in the response
- Used to retrieve metadata about the entity implied by the request without transferring the entity itself. (e.g. to test for link validity or resource modification.)
Like GET, it must be supported by all general purpose servers.
POST
- Perform resource, specific processing of the entities.
- Typically used in submitting HTML form data.
PUT
- Allows to upload data and store.
DELETE
- Allows to remove data in the storage.
OPTIONS
- allows the client to determine the options and/or requirements associated with a resource
TRACE
- Request a loop-back of the request message or echo back to the client.
- Used for testing or diagnostics of the request or response chain.
CONNECT
- Used only for HTTPS.

Hypertext Markup Language
