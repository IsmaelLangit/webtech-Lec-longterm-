LESSON 6:HTTP request methods

  • Standard Methods:

    • GET
      • - Most commonly used HTTP method
      - Must be supported by all compliant general purpose servers
    • HEAD
      • - MSame as GET except that the entity is not included in the response
      - Used to retrieve metadata about the entity implied by the request without transferring the entity itself. (e.g. to test for link validity or resource modification.)
      Like GET, it must be supported by all general purpose servers.
    • POST
      • - Perform resource, specific processing of the entities.
      - Typically used in submitting HTML form data.
    • PUT
      • - Allows to upload data and store.
    • DELETE
      • - Allows to remove data in the storage.
    • OPTIONS
    • TRACE
      • - Request a loop-back of the request message or echo back to the client.
      - Used for testing or diagnostics of the request or response chain.
    • CONNECT
      • - Used only for HTTPS.

  • Extended Methods:

    • e.g. WebDAV(RFC 4918)
      • - PROPFIND
      - PROPPATCH
      - MKCOL
      - COPY
      - MOVE
      - LOCK
      - UNLOCK
    • - SAFE
      • - Doesn't make any changes.
    • - IDEMPOTENT
      • - Can use multiple times but same result.
    • - CACHEABLE
      • - Response can be cache or not depend on others.
